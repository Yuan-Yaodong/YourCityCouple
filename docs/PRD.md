# 春节旅游目的地测试小程序 - 产品需求文档

## 1. 项目概述

### 1.1 项目背景
参考爆款案例「测测你最旺的城市」，打造一款春节旅游目的地测试小程序。用户通过回答趣味问题，获得最适合自己去旅游的城市结果。

### 1.2 核心玩法
- 用户进入小程序 → 回答5道趣味问题 → 生成个性化旅游推荐结果
- 结果包含：推荐城市、城市介绍、适合玩法、为什么适合你

### 1.3 目标用户
- 18-35岁年轻人
- 春节期间有旅游计划或对旅游感兴趣
- 喜欢社交分享

---

## 2. 功能规划

### 2.1 核心功能

| 功能 | 描述 |
|------|------|
| 引导页 | 展示标题、副标题、开始按钮 |
| 问题页 | 展示问题+4个选项，用户点击选择 |
| 结果页 | 展示推荐城市、详情介绍、分享按钮 |
| 分享功能 | 生成分享图片/小程序卡片 |

### 2.2 问题设计（5道题）

每道题4个选项，对应不同的城市属性维度：

| 题号 | 问题 | 维度 |
|------|------|------|
| Q1 | 新年第一顿想吃啥？ | 美食偏好 |
| Q2 | 偏好哪种旅行方式？ | 节奏偏好 |
| Q3 | 喜欢什么天气？ | 气候偏好 |
| Q4 | 旅行中最在意什么？ | 体验偏好 |
| Q5 | 用哪个词形容新年？ | 心情偏好 |

### 2.3 结果城市库（12个城市）

| 城市 | 特征 | 目标用户 |
|------|------|----------|
| 哈尔滨 | 冰雪世界、浪漫 | 喜欢冬季、浪漫 |
| 三亚 | 温暖海滨、度假 | 怕冷、度假 |
| 成都 | 美食、悠闲 | 吃货、慢生活 |
| 西安 | 古都、历史文化 | 文化、历史 |
| 丽江 | 古城、艳遇 | 文艺、放松 |
| 厦门 | 海岛、小清新 | 文艺、浪漫 |
| 杭州 | 西湖、江南 | 山水、诗意 |
| 青岛 | 海滨、啤酒 | 休闲、海鲜 |
| 雪乡 | 童话、冰雪 | 体验型、拍照 |
| 大理 | 苍山洱海、风花雪月 | 疗愈、放松 |
| 广州 | 美食、粤文化 | 吃货、现代 |
| 重庆 | 火锅、山城 | 刺激、美食 |

### 2.4 计分逻辑

每个选项对应不同的城市属性权重：
- Q1美食偏好：川菜→成都/重庆，粤菜→广州，海鲜→青岛/厦门，东北菜→哈尔滨/雪乡
- Q2旅行方式：打卡→西安/厦门，度假→三亚/丽江，冒险→哈尔滨/重庆，随性→大理/杭州
- ...以此类推

最终统计各城市得分，取最高分为结果。

## 3. UI/UX 设计要求

### 3.1 视觉风格
- **主色调**：春节红 (#E62E2E) + 金色 (#FFD700)
- **辅助色**：暖黄 (#FFF8E1)、福字红 (#C41E3A)
- **风格**：喜庆、吉祥、有年味

### 3.2 字体
- 标题：粗体、圆润
- 正文：清晰易读

### 3.3 动画效果
- 题目切换：滑动动画
- 选项点击：缩放反馈

## 4. 技术方案（简要）

### 4.1 技术栈
- **框架**：微信小程序原生开发
- **状态管理**：简单全局状态
- **数据存储**：本地存储（答案、结果缓存）

### 4.2 目录结构

```
miniprogram/
├── app.js
├── app.json
├── app.wxss
├── pages/
│   ├── index/          # 引导页
│   ├── quiz/           # 问题页
│   └── result/         # 结果页
├── components/         # 公共组件
├── utils/              # 工具函数
```
